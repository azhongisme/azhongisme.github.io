树状数组
2023/8/2

树状数组顾名思义就是一个数组长的像树，树状数组主要涉及的操作为求前缀和和修改某一个数

求前缀和

int lowbit(int x){
    return x & -x;
}
int sum(int x){
    int res = 0;
    for(int i = x;i;i-=lowbit(i)) res += tr[i];
    return res;
}

修改a[x],将a[x] += c

void add(int x,int c){
    for(int i = x;i<=n;i+=lowbit(i)) tr[i] +=c;
}

上面两个操作中的+-lowbit(i)分别为求父节点和子节点
树状数组的原理主要就是二进制 假设k表示x的二进制表示下最后一位1(….1000…00),有k个0，c[x]表示以x结尾，长度为2^k的区间和
这里以16为例：16的二进制表示为10000，首先将c[x] = a[x],那么接下来就是01111，将其分为四个区间(均为左开右闭)
(01110,01111],(01100,01110],(01000,01100],(00000,01000]也就是说c[16] = a[16] + c[15] + c[12] + c[8]

并查集
2023/7/31

并查集是一种用于管理元素所属集合的数据结构，实现为一个森林，其中每棵树表示一个集合，树中的节点表示对应集合中的元素。
并查集主要有两种操作：（1）合并两个集合 （2）查询某个元素所在的集合（根节点）
并查集的优化方式也是有两个：一个为路径压缩，宁一个为按秩合并(启发式合并)。两种优化方式最终都是O(logn)的复杂度，同时使用路径压缩和启发式合并之后，并查集的每个操作平均时间仅为 O(\alpha(n))，其中 \alpha 为阿克曼函数的反函数，其增长极其缓慢，也就是说其单次操作的平均运行时间可以认为是一个很小的常数。
我们还可以在并查集的边上定义某种权值、以及这种权值在路径压缩时产生的运算，从而解决更多的问题。带权并查集的思想可以理解成一种维护元素间的相对关系。
并查集还有一种是扩展域，采用的是枚举的思想。
